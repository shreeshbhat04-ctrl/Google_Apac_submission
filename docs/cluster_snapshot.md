# Cluster Snapshot Before Teardown

This file preserves the exact AlloyDB state that was confirmed before the
cluster was deleted for billing reasons.

Use this as the source of truth when recreating the environment later.

## Cluster Context

- Project: `mystical-app-490317`
- Region: `us-central1`
- Cluster: `my-alloydb-cluster`
- Instance: `my-primary-inst`
- Database used for testing: `my_application_db`
- Network mode: private IP only
- Auth path validated for SDK work: IAM auth via `AsyncConnector` + `asyncpg`

## Confirmed Database Flags

```sql
SELECT name, setting
FROM pg_settings
WHERE name IN (
  'google_ml_integration.enable_ai_query_engine',
  'google_ml_integration.enable_model_support'
)
ORDER BY name;
```

Returned:

```text
google_ml_integration.enable_ai_query_engine | on
google_ml_integration.enable_model_support   | on
```

## Confirmed Built-In Model Availability

`google_ml.model_info_view` returned Google-available built-in embedding models,
including:

- `text-embedding-005`
- `gemini-embedding-001`
- `text-multilingual-embedding-002`

Implementation implication:
- `text-embedding-005` can be treated as available by default in this environment.

## Confirmed Embedding Dimension

```sql
SELECT vector_dims(google_ml.embedding('text-embedding-005', 'hello world')::vector);
```

Returned:

```text
768
```

Implementation implication:
- `embedding vector(768)` is the correct v1 schema choice for this environment.

## Confirmed Products Schema

The validated table shape was:

```sql
CREATE TABLE products (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    price NUMERIC(10, 2),
    category VARCHAR(100),
    metadata JSONB NOT NULL DEFAULT '{}'::jsonb,
    embedding vector(768),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## Confirmed Vector Search

```sql
SELECT id, name, category, price,
       embedding <=> google_ml.embedding('text-embedding-005', 'running shoes')::vector AS distance
FROM products
ORDER BY distance
LIMIT 5;
```

Returned:

```text
1 | Test product | shoes | 79.99 | 0.2051991270772745
```

## Confirmed Filtered Vector Search

```sql
SELECT id, name, category, price,
       embedding <=> google_ml.embedding('text-embedding-005', 'running shoes')::vector AS distance
FROM products
WHERE category = 'shoes'
  AND price <= 100
ORDER BY distance
LIMIT 5;
```

Returned the same row successfully under filter constraints.
